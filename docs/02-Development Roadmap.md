# Chapter 2 - Development Roadmap

Purpose:
This chapter defines how the BookVerse project will be developed, including the project phases, milestones, deliverables, estimated timeline, and development methodology. It acts as the project's execution plan and ensures that development is organized, traceable, and aligned with the project's objectives.

## Chapter Overview

## 2.1 Development Methodology

The BookVerse project will follow an iterative and incremental development approach inspired by Agile software development principles. Instead of developing the entire application at once, the project will be divided into smaller phases called sprints. Each sprint will focus on completing a specific module, allowing continuous testing, documentation, and improvement throughout the development lifecycle.

Every development cycle will include the following activities:

- Requirement Analysis
- System Design
- Implementation
- Testing
- Documentation
- Git Commit
- Code Review
- Interview Preparation

This methodology allows better project management, easier debugging, and continuous learning while ensuring that every module is completed before moving to the next.

## 2.7 Risk Assessment

Potential risks identified during development include:

- Learning curve for Spring Security and JWT.
- Integration issues between Angular and Spring Boot.
- Database design modifications during development.
- Deployment and server configuration challenges.
- Time constraints due to project complexity.

These risks will be minimized through incremental development, regular testing, documentation, and continuous code review.

## 2.8 Assumptions and Constraints

### Assumptions

- Development will be carried out by a single developer.
- Required software tools are installed and configured.
- Internet connectivity is available for research and dependency management.

### Constraints

- Development timeline is limited to approximately six weeks.
- Initial version will focus only on core features.
- Payment gateway integration will be implemented in future versions.

## 2.10 Success Metrics

The project will be considered successful if:

- All planned modules are completed.
- The application functions without critical defects.
- REST APIs are secure and properly documented.
- Angular frontend integrates successfully with the backend.
- Documentation remains complete and up to date.
- The project demonstrates enterprise software development practices.
- The developer can confidently explain the architecture, implementation, and design decisions during technical interviews.
